# Command Center Structure

This repository is organized into a few clear layers:

- `src/`, `public/`, `prisma/`: the actual application
- `ops/`: operating system, planning, priorities, workflows, and OpenClaw instructions
- `research/`: SEO research, market analysis, business cases, and project discovery
- `docs/`: governance, reference docs, and system documentation
- `data/`: reference data and licensed/static input material
- `team/`: team-related material and role references
- `archive/`: historical or temporary material not part of active operations
- `content/`, `templates/`, `shared/`: reusable delivery assets

## Operating rule
The repo should act as a single source of truth. Root clutter is avoided on purpose.
