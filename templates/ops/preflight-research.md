# Pre-flight — Research (Wout) ✅

**Rule:** geen pre-flight = geen review.

## Scope & defaults
- [ ] Doel (1 zin) ingevuld
- [ ] Regio/landen gekozen (bv. BE-VLG + NL)
- [ ] Taal gekozen (NL-only / NL+FR)
- [ ] NIET in scope (max 3 bullets)

## SERP & page type
- [ ] `serp_type` bepaald (local pack / organic / mixed)
- [ ] `page_type` gekozen (city/service/tool/blog) + 1 zin waarom

## Inputs voor build (geen interpretatie)
- [ ] Keyword sheet in vast format aanwezig (kolommen vast)
- [ ] Content model JSON aanwezig (velden/kolommen die builder rendert)
- [ ] URL/slug rules vastgelegd
- [ ] Internal linking rules (max links + anchors) vastgelegd

## CTA & offer
- [ ] Primary CTA + secondary CTA ingevuld
- [ ] Proof/Trust block(s) beschreven (wat tonen we?)

## Edge cases / exclusions
- [ ] Do-not-target / brand risk / legal notes ingevuld
- [ ] Missing data handling (wat als X ontbreekt?)

## DoD
- [ ] Acceptance criteria (DoD) checklist afgevinkt (link naar `ops/DoD.md`)

## Output
- [ ] 1 entrypoint file: `projects/YYYY-MM-DD_<slug>/handoff.md` met links naar alle assets
