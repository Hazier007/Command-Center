# Team Workflow

## Core flow

### 1. Intake -> Brief
For every request, create:
- Context
- Goal
- Scope (in / out)
- Deliverable
- Acceptance criteria
- Risks / dependencies
- First next action

### 2. Triage -> Assign
**Command Center first. Not chat-first.**

Ownership model:
- **Lisa ??** -> strategy / ops / decisions / final review
- **Wout ??** -> research / SEO / data
- **JC ??** -> build / code / infra

### 3. Execute -> Control -> Ship
- Lisa guards deadlines and dependencies
- Lisa does final review
- output is only done when it is working and tested

## Rules
- no false reporting
- no passive execution
- assumptions must be labeled
- default to action
- Command Center is the source of truth

## Handoffs
- Wout produces evidence, findings, opportunities, and recommendations
- JC produces implementation, fixes, and tested output
- Lisa decides, sequences, reviews, and ships
