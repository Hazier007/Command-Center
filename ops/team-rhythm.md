# Team Rhythm

## Default rhythm
- Keep priorities explicit.
- Every active workstream needs a next action or a blocker.
- Nothing should float unowned.

## Cadence
- Priorities reviewed whenever strategic direction changes.
- Status reviewed whenever an important workstream moves or blocks.
- Backlog updated when new opportunities or parked work appears.
- Decisions logged when a default, rule, or operating choice changes.

## Command discipline
- Capture important work in Command Center.
- Avoid managing work through scattered chat history.
- Use the repo to preserve continuity after technical issues or session loss.
